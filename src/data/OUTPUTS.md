# Radar Compras Output Documentation

Complete guide to interpreting the JSON output files generated by `trend_extractor_v2.py`.

## Overview

The extractor generates **6 JSON files** containing aggregated procurement statistics from OCDS Guatemala data. Each file is optimized for web charting and analysis.

**Data Period:** 2026-01-02 to 2026-04-08

## 1. summary.json

**Purpose:** Global statistics and metadata about the entire dataset.

### Schema

```json
{
  "generated_at": "ISO 8601 timestamp",
  "period": {
    "start": "YYYY-MM-DD",
    "end": "YYYY-MM-DD"
  },
  "totals": {
    "processes": "integer (procurement processes)",
    "unique_buyers": "integer (government entities)",
    "unique_suppliers": "integer (vendor companies)",
    "total_amount_gtq": "float (total spending in GTQ)",
    "total_items": "integer (sum of all item quantities)"
  }
}
```

### Example

```json
{
  "generated_at": "2026-05-19T05:40:03.147264Z",
  "period": {
    "start": "2026-01-02",
    "end": "2026-04-08"
  },
  "totals": {
    "processes": 143233,
    "unique_buyers": 546,
    "unique_suppliers": 7940,
    "total_amount_gtq": 50222443389.64,
    "total_items": 1948387713
  }
}
```

### Use Cases

- **Dashboard header:** Display key metrics
- **Context:** Report generation and data validation
- **Timestamp verification:** Confirm when data was last updated

## 2. top_buyers.json

**Purpose:** Top 10 government entities by procurement process count, with monthly trends.

### Schema

```json
{
  "rank": [
    {
      "rank": "integer (1-10)",
      "name": "string (entity name)",
      "total_processes": "integer (cumulative count)",
      "pct_of_total": "float (0-100)",
      "monthly_series": [
        {
          "month": "YYYY-MM",
          "processes": "integer"
        }
      ]
    }
  ]
}
```

### Example (Rank #1: IGSS)

```json
{
  "rank": 1,
  "name": "INSTITUTO GUATEMALTECO DE SEGURIDAD SOCIAL -IGSS-",
  "total_processes": 92460,
  "pct_of_total": 64.55,
  "monthly_series": [
    { "month": "2026-01", "processes": 16106 },
    { "month": "2026-02", "processes": 21406 },
    { "month": "2026-03", "processes": 43246 },
    { "month": "2026-04", "processes": 11702 }
  ]
}
```

### Insights

- **IGSS dominates:** 64.55% of all procurement processes (92,460)
- **March peak:** Spike to 43,246 processes in 2026-03
- **April drop:** Sharp decline to 11,702 in 2026-04 (partial month)

### Use Cases

- **Line/area charts:** Show IGSS monthly trend vs. competitors
- **Bar charts:** Rank top 5-10 buyers by total processes
- **Trend analysis:** Identify seasonal patterns
- **Supplier targeting:** Where 2/3 of government spending goes

## 3. top_suppliers.json

**Purpose:** Top 10 private vendors by number of awarded contracts, with monthly adjudication trends.

### Schema

```json
{
  "rank": [
    {
      "rank": "integer (1-10)",
      "name": "string (vendor name)",
      "total_adjudicaciones": "integer (contracts awarded)",
      "pct_of_total": "float (% of all suppliers)",
      "monthly_series": [
        {
          "month": "YYYY-MM",
          "adjudicaciones": "integer"
        }
      ]
    }
  ]
}
```

### Interpretation

- **total_adjudicaciones:** Number of times this supplier won a contract
- **adjudicaciones (monthly):** Contracts awarded in that month
- **pct_of_total:** % of ALL contracts across ALL suppliers

### Use Cases

- **Market concentration:** Identify monopolies and dominant vendors
- **Vendor trend analysis:** Track supplier growth/decline monthly
- **Risk assessment:** Vendors with >10% market share

## 6. item_descriptions.json

**Purpose:** Top 20 product/service descriptions by purchase frequency (unit count).

### Schema

```json
{
  "items": [
    {
      "rank": "integer (1-20)",
      "description": "string (product name, truncated 100 chars)",
      "count": "integer (total units purchased)",
      "pct_of_total": "float (% of all items purchased)"
    }
  ]
}
```

### Example

```json
{
  "rank": 1,
  "description": "MEDICINAS INYECTABLES",
  "count": 5234,
  "pct_of_total": 3.42
}
```

### Interpretation

- **count:** Total quantity of this item purchased across all processes
- **pct_of_total:** % of all items purchased (sum across all descriptions)
- **Difference from top_items_amount:** This ranks by **volume**, not by spending amount

### Use Cases

- **Popular items:** What does the government buy most of (by volume)?
- **Supply chain planning:** High-demand items
- **Supplier management:** Items most frequently procured

## Data Quality Notes

### Date Range Considerations

- **Period:** 2026-01-02 to 2026-04-08 (4+ months)
- **April incomplete:** Only 8 days of data (2026-04-01 to 2026-04-08)
- **Caveat:** Monthly comparisons should note April is partial

### Item Counts vs. Spending

**Why item_count can be huge (e.g., 8M units) but adjudication count is small:**

When a single procurement process awards multiple items, the script allocates the total spending proportionally across items:

```
Process total: Q100M
Items: 2 (Road + Labor)
  - Road: 500 units → 500/1000 * Q100M = Q50M
  - Labor: 500 units → 500/1000 * Q100M = Q50M
```

This means:
- **top_items_amount.json**: Units allocated proportionally
- **top_suppliers.json**: Contract count (one award = one line in adjudicaciones)
- **item_descriptions.json**: Sum of units for each description

### Missing Data Handling

- Records with no `publishedDate` → month = "unknown"
- Records with no awards → no spending/items recorded
- Records with no items → counted as process but no item breakdown

## Charting Integration

### Chart.js Examples

#### Line Chart: Top Buyer Trend

```javascript
const response = await fetch('outputs/top_buyers.json');
const data = await response.json();
const igss = data.rank[0];

new Chart(ctx, {
  type: 'line',
  data: {
    labels: igss.monthly_series.map(m => m.month),
    datasets: [{
      label: igss.name,
      data: igss.monthly_series.map(m => m.processes),
      borderColor: '#c0392b',
      fill: false
    }]
  }
});
```

#### Pie Chart: Spending by Category

```javascript
const response = await fetch('outputs/top_categories.json');
const data = await response.json();

new Chart(ctx, {
  type: 'pie',
  data: {
    labels: data.categories.map(c => c.segment_name),
    datasets: [{
      data: data.categories.map(c => c.pct_of_total),
      backgroundColor: ['#3498db', '#e74c3c', '#2ecc71', ...]
    }]
  }
});
```

#### Bar Chart: Top 5 Buyers

```javascript
const response = await fetch('outputs/top_buyers.json');
const data = await response.json();
const top5 = data.rank.slice(0, 5);

new Chart(ctx, {
  type: 'bar',
  data: {
    labels: top5.map(b => b.name),
    datasets: [{
      label: 'Procesos',
      data: top5.map(b => b.total_processes),
      backgroundColor: '#3498db'
    }]
  }
});
```

## Common Questions

**Q: Why is April data so different?**  
A: April only contains data through 2026-04-08 (8 days). Comparisons should note partial month.

**Q: Why are item_counts sometimes decimal-looking?**  
A: They've been rounded to integers. Decimal values occur during proportional allocation.

**Q: How are monthly_series ordered?**  
A: Chronologically by YYYY-MM string (natural sort order).

**Q: Can I drill down into individual processes?**  
A: No, the extractor only outputs aggregates. For detail, query the source JSONL directly.

**Q: How often should this be run?**  
A: Set up a monthly cron job after the 5th of each month (to capture full month data):
```bash
0 2 5 * * /usr/bin/python3 /path/to/trend_extractor_v2.py "data/guatemala_bulk_2026.jsonl/*.jsonl" --output outputs
```
